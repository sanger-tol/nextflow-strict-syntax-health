        Typical pipeline command:

  nextflow run sanger-tol/genomeassembly -profile <docker/singularity/.../institute> --input samplesheet.csv --outdir <OUTDIR>

help message of that parameter will be printed.  
or `--helpFull`.  

Input/output options
  --genomic_data                        [string] Input YAML specifying the location of all genomic data for assembly. See the documentation for a 
full specification.  
  --assembly_specs                      [string] YAML file specifying the parameters for a set of assemblies, including datasets and arguments. See 
the documentation for a full specification.  
  --outdir                              [string] The output directory where the results will be saved. You have to use absolute paths to storage on 
Cloud infrastructure.  
  --email                               [string] Email address for completion summary. 

Input preparation options
  --kmer_size                           [integer] Kmer size to use for all kmer-based subroutines in the Kmer profiling subworkflow. This parameter is 
overridden by the provided kmer size for a given input read set if a FastK database is provided. 
[default: 31]  
  --genomescope_args                    [string]  Optional command-line arguments to pass to genomescope2. 
  --save_fastk_databases                [boolean] Save the FastK databases generated by the pipeline to the output directory. 

Purging options
  --purging_reads_chunk_size            [integer] How many reads to map in a single chunk when mapping reads to the primary assembly during purging. 
[default: 500000]  
  --ont_minimap2_map_mode               [string]  Which minimap2 mapping mode to use when mapping ONT? See 
https://lh3.github.io/minimap2/minimap2.html for a full list. [default: map-ont]  

Polishing options
  --polishing_longranger_container_path [string]  Path to a container with longranger to use for the longranger processes. Required for polishing to 
run.  
  --polishing_n_chunks_bed              [integer] Number of chunks to split assembly into for polishing. [default: 100] 

Hi-C mapping and scaffolding options
  --hic_aligner                         [string]  Which aligner to use to align Hi-C reads to the reference  (accepted: bwamem2, minimap2) 
[default: bwamem2]  
  --hic_mapping_cram_chunk_size         [integer] Number of CRAM slices to include per CRAM file per mapping chunk. Lower values mean more Hi-C 
mapping jobs are spawned, but each should have a lower run time. [default: 10000]  
  --hic_mapping_minq                    [integer] Minimum MapQ score for a Hi-C read to be retained for scaffolding. [default: 10] 
  --scaffolding_cool_bin_size           [integer] Bin size to use when building cooler contact maps. [default: 1000] 

Assembly QC options
  --busco_lineage_directory             [string] Path to a local directory containing pre-downloade BUSCO lineages. 

Organelle assembly options
  --mitohifi_min_ref_len                [integer] The minimum size of a reference organellar genome to use in reference-based organelle assembly with 
Mitohifi [default: 15000]  

Generic options
  --help                                [boolean, string] Display the help message. 
  --help_full                           [boolean]         Display the full detailed help message. 
  --show_hidden                         [boolean]         Display hidden parameters in the help message (only works when --help or --help_full are provided).
